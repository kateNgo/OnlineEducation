Purchase Orders

 

PURCHASE ORDERS

 

 

If you are using alternate stock in the correct manner, the purchase order PLR window will automatically select the correct alternate part number for the supplier the purchase order is being created for.

 

There is now an option to auto create an alternate stock code equivalent to the main code and also to not increment or decrement alternate stock so it can be used merely as a search criteria.

 

If information has been entered into the Status Field (salesperson) in Purchase Orders, the salespersons initials now print on the header of the order.

 

If you are using Barcoding, when you receive a purchase order, the system will automatically prompt to print the stock labels immediately.

 

We now have a tick box in the Others>control system>Company details and under the security options tab under Suppliers, to print prices on purchase orders, so the tick box doesnalways have to be ticked in the print option screen.

 

In Reporting>Stock Menu, the Orders & Deliveries report now has the ability to print the details in the report between two dates.

 

You can now select a different purchase order format than the invoice format. You have 9 formats to choose from. Select the format by going into Others>Control System>Printer Settings.

 

Purchase orders now allow you to set an order due date and then allows you to filter a purchase order by due date. Simply make sure you tick the box on the Show Purchase Orders Screen which says Use Due Date NOT date.

 

There is now an option to assign a purchase order limit to an employee which must be approved by an employee with an adequate limit for the selected order.

 

You can now track the staff member who Created, last modified, received, locked or last printed a purchase order by the initials which appear in the relevant fields on the purchase order.

 

You now have a control file option to force a description and the status to be selected before saving a purchase order.

 

There is now an import button to allow a CSV import into a purchase order. The format of the CSV file is Part Number, Description, price (optional)

There is now a clone to button which allows a purchase order to be cloned to another supplier, a new job or a new sales order for the same supplier or different. This is excellent for re-occurring orders.

 

There is now a levy button on a purchase order which allows a levy to be added to the order.
