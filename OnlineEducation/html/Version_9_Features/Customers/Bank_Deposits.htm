Bank Deposits

There is a Deposits Reconciliation which allows you to reconcile the cash on hand with moneys received. To use this function, Go to the Ledger menu and select Deposits. This will bring up the Deposits screen.

Click on the Type Field and choose either Posted or Un Posted. This is generally defaulted to Unposted.

You must then select the till/cash account for the deposit to be reconciled from. You can select any cash account between 1010  1019 or you can select ALL which reconciles all of the cash accounts at once. Select the account by clicking on the Arrow next to the Cash Acc field.

You can then select the date that the deposit is for, it could be today or any previous date. You cannot post forward.

You must then select the bank account to post the deposit into. Click on the Arrow next to the Post field to select the bank account. Bank accounts range between 1020 - 1029.

Toggle - The Toggle button is used to tag an entry which you want to post or reverse. Highlight the entry by clicking on it once, and then click on the Toggle button. A green tick should now appear next to the entry. You can also TAG an entry by double clicking on it.

Select All - The Select All button is used if you want to tag all the entries, but you donwant to have to tag them all each individually. By pressing the Select All button, all entries will be tagged automatically.

Unpost Reverse - The Reverse button is used to move an entry from to (and back again) without actually posting or unposting the entry. For example, if you have entries in this section which have already been posted using a journal entry, and you just need to move them over to the posted section without actually posting them a second time, you could use the reverse button.

To Reverse an entry (either in the posted or unposted section) you must tag the entry, and then click the Unpost Reverse button. Once the Reverse button is pressed, the screen will be cleared and the entry would have moved to the opposite section.

Note: The system will notify you if you accidentally click on the Reverse button without tagging an item first.

Posting an Entry

Highlight the entry/entries to be posted. You must ensure that the cash amount shown at the bottom of the screen at the Total Cash A/c Only field is correct. If not then you must manually type in the correct cash amount to be deposited. You must also check that the total amount to be deposited is correct.

Then Click on the Post button. This will transfer the money from the selected cash account/s into the selected bank account and then print a copy of the deposit slip.

If you have not set up your bank account details then the account number, name and BSB number will be blank. You are required to set up these details in the General Ledger masterfile where you can select the bank account you will be using, and then clicking on the Update Bkrec button and inserting the bank account details in the option box.

Dissect Button.

If you want to dissect the deposits into various accounts then click on the Dissect Button. A new screen will appear with fields for different accounts.

Click into the Amount fields for each account you want to post money into, and type in how much money you want to post into that account. The Total field will display the amount you have posted so far. The Total Amount Tagged field will display the amount which you tagged to be posted. The Left To Post field will display the amount you have left to post in the different accounts.

The system will notify you if you enter amounts which exceed the amount you have left to post.

Once you have completed entering the amounts into the accounts, click on the Process button (just click the Close button if you donwish to process it).

A Process Completed window will appear. Click the OK button.

The screen will then clear, and the entries will now appear in the Posted section.

PLEASE NOTE: Once an entry is in the posted it cannot be unposted.

The TOGGLE, REVERSE and SELECT ALL buttons all work the same in the type of Posted as they did in UnPosted.

 

 

 

Deposit Rules

There are some specific rules that determine how the deposit screen gathers and processes its information. If a cheque number is the same for the same day then the system assumes that even though there is more than one entry for the cheque number on that day there was really only one cheque and so it adds up all of the entries that are the same and makes it into one cheque to be deposited. This allows you to receive one cheque which is used to pay more than one account and have the deposit screen process it as one cheque.

This may occur if for example someone pays a cheque to cover both his own account as well as his wifeor childaccount. You need to record two or more payments but the deposit sees it as one cheque. If you see a deposit which says many then this is what has happened. This will cause a problem if you do not use correct cheque numbers and just use a fullstop or some other character or name against the cheques to deferintiate each entry.

One other advantage to this is that if you use a cheque number called EFT for all of your manual credit card transactions then they will be treated as one total amount as they are on most deposit slips. When used correctly this is a good feature. However, if not fully understood it can be a nightmare to some users.

Please Note: that any entries classed as CASH or any entries journalled into the 1010 account with no method will be treated as CASH even if it wasnactually cash.
