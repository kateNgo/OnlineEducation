Customer Membership Details

 

Am-win now allows for the setup and use of allowing customers to accumulate Membership points in a similar fashion to how you can accumulate points with your credit card. A customer can be allocated a Membership number, which can be up to 10 characters long and can be manually entered or barcoded into the field. As the customer purchases from you, points are accrued and can be available at a later date to be used against a customer invoice for an item which they can select using their accrued points.

The setup allows for a certain number of points to be accrued per dollar spent, and the value of accrued points will display on the screen, and will display on the customers invoice if the tick box in the control file allows this. Also if the customers account has exceeded their trading terms, there is a tick box in the control file to not allow redemption of points for invoices which are not yet paid for.

These tick boxes are in the Business Options>Invoice Tab, Print Accrual points on Invoice, and Job Payments - Make Points Amount Zero if Terms overdue.

Print Accrual points on Invoice, and Job Payments  means that the accrual points will be printed and displayed on each invoice to show a current up to date point value.

Make Points Amount Zero if Terms overdue  means that if the invoices are overdue, the accrual points do not apply.

 

Membership No  This is where you can enter or barcode the Membership number into the Membership No field. This is useful if you already have a membership points system in place, you can enter the same membership number currently in use. Alternatively, it can be whatever number you choose to allocate to that particular customer.

 

Expiry  This is the expiry date for the Membership card. Only enter the month and year. If the membership has expired then the member will no longer accumulate points.

 

Password - This field is used to record a password against the member. This password can be used to verify the member.

 

Drivers Licence  This is where you can enter in the customers Drivers Licence number, then Drivers Licence expiry date.

 

Date of Birth  This is where you can record the birthdate of your customer. This information is used for identification purposes.

 

Member Type  Select whether your customer will be a Gold, Silver or Bronze member. You can alter the membership types from the attributes file if you wish to use a different method of dissecting memberships.

 

Status  When adding Membership details the Status will automatically be Active. If required, the Status can be changed to Expired, Cancelled or Suspended. If the member is not active then they will no longer accumulate points.

 

Points Per Dollar  This field contains the number of points that will be accrued per dollar purchased. Points are accrued through the invoicing process and reduced by either a credit return or by the member using their points to pay for goods and services through the payments area.

 

Points Accrued  This field contains the number of Points Accrued so far for the selcted customer.

 

Value per Point  This field shows the value that each point is worth when the points are redeemed by the customer through the payments area.

 

Points Value $ - This field will display the dollar value of points accrued which is equal to the number of points accrued times the value per point.

 

Details:

In the set up area under the customer masterfile in the Membership tab you can nominate how many points are accumulated by the customer for each dollar spent. This usually will default to one point per dollar. When an invoice is produced, the number of points accumulated is equal to the value of the invoice including GST, multiplied by the points per dollar set up in the customer masterfile. Using the default values; a $100 invoice would generate 100 points for the customer. These points can then be redeemed to pay for services or goods that the customer has purchased.

 

Membership points can be accumulated via a number of processes in the system. These processes include job invoicing, customer orders, consignment stock and POS invoicing. Points will be reduced via credit returns.

The system allows for a certain number of points to be accrued per dollar spent with the value of accrued points displayed on the screen.

 

The points accrued can display on the customers invoice if the tick box in the control file allows this. In addition to this, if the customer has exceeded their trading terms or the customer is on credit hold, there is a tick box in the control file to not allow redemption of points for invoices which are not yet paid for.

These tick boxes are in the Business Options>Invoice Tab, Print Accrual points on Invoice, and Job Payments - Make Points Amount Zero if Terms overdue.

 

Sales order (asset) invoices do not accumulate points as typically there is not sufficient margin in the sale of an asset to accumulate membership points for the customer.

 

When points are accumulated by an invoice or reduced via a credit return, the pointliability which is based on the number of points multiplied by the value per point set up in the customer masterfile is allocated to 2997 membership points ledger which is a system controlled ledger and automatically created when AMWIN is started.

This ledger accumulates the value of the liability of the points owing to all customers. The other side of this entry is created as an expense number 9997 in a membership expense ledger.

 

Points that are accumulated on the current invoice cannot be redeemed against an invoice through the customer invoice payment window. A customer can only redeem points to the maximum available points and as a result points can never be redeemed into a negative value. When making a payment via invoicing, the customer is able to redeem points and pay the balance by up to two additional payment methods at the same time.

 

The payment methods screen shows the current point accrued and the available dollar value for redemption. On selecting POINTS as a payment method, the value redeemed is the invoice amount up to the value of the available pointvalue. Points cannot be redeemed for cash and points can never be redeemed into a negative position.

 

The payment method is noted as POINTS for the customer to see. Points can be redeemed from the customer payments screen in a similar fashion.

 

Membership Points Report

Under Reports Customer / Supplier > Cust Sup Listing & Cus Source > Membership is a basic membership report that shows the members by group and membership type. The report can also be filtered by pointvalue to show the points exposure.

 

Initial setting of customer membership

In the AMWIN Tools Program you can mass set up membership details for customers. This program will allow you to set up all membership criteria based on the customer group, territory or Mail Group.

 

The tool allows you to nominate one or more of these criteria and then create the membership data for the selected range of customers.

 

An additional tool allows you to auto generate customer membership points based on previous sales from a particular date. This is useful if you started using the membership process mid financial year but wanted to generate customer points from any invoices or credits done for that customer from the beginning of the financial year or another nominated date period even if this was the beginning of the current month if you started membership points mid month.

 

This tool creates the appropriate liability and cost of sales transactions for the membership points that are generated.
