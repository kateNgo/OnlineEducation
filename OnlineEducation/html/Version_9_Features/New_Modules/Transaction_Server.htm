Transaction Server

 

One of the issues of any database management software, is the inability to update a database when a record is held by another user. In AMWIN, there are a number of processes that update a job or manufactured item externally. The most common example of this is when items are allocated from a purchase order to a job. If a part is allocated to a job from an order the item is placed onto the job at the time the order is either received or received and finalised. However, if the job is currently held open by another user, the purchase order process is not able to update the job and you will be warned that the purchase order was unable to do so. This would require that you either wait until the other user exits from the job or continues to receive the order and manually add the parts to the job at a later stage.

 

When the Transaction server is activated, and the job is held open, while another process is trying to update it, AMWIN will automatically post these records into a temporary database file. The Transaction Server then automatically monitors this temporary file and will update the job or manufactured item as soon as it becomes available by the other user exiting from the job. The Transaction server will continue to monitor this process to ensure that all parts, labour and revenue items are allocated to a job, end up on the job. In the event that the user opening the job does not exit the job but in fact finalises it, the user will be notified that items being allocated to that job are not currently on the job, at which time these additional items will automatically be placed on the job ensuring they are not missed when the invoice is finalised.

 

Other processes that also automatically update the invoice via the Transaction server include the Timesheet Labour Module and the AM Parts Allocation module.

 

The Transaction server is an essential program that has to be run together with all Timesheet, purchase order and AM-Parts transactions. The Transaction Server must be included in the scheduled programs to be run upon startup of the computer. It should be always running in the background.
