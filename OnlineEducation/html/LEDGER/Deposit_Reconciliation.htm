Deposit Reconciliation

 

There is a Deposits Reconciliation which allows you to reconcile the cash on hand with moneys received.

 

Go to the Ledger menu and select Deposits. This will bring up the Deposits screen.

 

Click on the radio button next to the Selection field and choose either Posted or Un Posted.

 

You must then select the till/cash account for the deposit to be taken from. You can select any cash account between 1010  1019 or you can select all which reconciles all of the cash accounts at once. Select the account by clicking on the arrow key next to the Cash Acc field.

 

You can then select the date that the deposit is for which could be today or any previous date.

 

You must then select the bank account to post the deposit into. Click on the arrow key next to the POST field to select the bank account. Bank accounts range between 1020  1029.

 

The TOGGLE button: The Toggle button is used to tag an entry which you want to post or reverse. Highlight the entry by clicking on it once, and then click on the Toggle button. A red dollar sign should now appear next to the entry.

 

You can also TAG an entry by double clicking on it. A red dollar sign will appear next to the tagged entry.

 

The SELECT ALL button: The Select All button is used if you want to tag all the entries, but you donwant to have to tag them each individually. By pressing the Select All button, all entries will be tagged automatically.

 

The U-REVERSE button: The reverse button is used to move an entry from unposted to posted (and back again) without actually posting or unposting the entry. For example, if you have entries in this section which have already been posted using a journal entry, and you just need to move them over to the posted section without actually posting them a second time, you could use the reverse button.

 

To Reverse an entry (either in the posted or unposted section) you must tag the entry, and then click the U-Reverse button.

 

Once the Reverse button is pressed, the screen will be cleared and the entry would have moved to the opposite section. The system will notify you if you accidentally press the Reverse button without tagging an item first.

 

Posting an Entry: highlight the entry/entries to be posted.

 

You must ensure that the cash amount shown at the bottom of the screen is correct and if not then you must manually type in the correct cash amount to be deposited. You must also check that the total amount to be deposited is correct.

 

Tag these entries using the Toggle button, on double clicking on the entry.

 

Click on the Post button. This will transfer the money from the selected cash account/s into the selected bank account and then print a copy of the deposit slip. If you have not set up your bank account details then the account number, name and bsb number will be blank. You are required to set up these details in the Bankrec module under the bank maintenance menu option.

 

A new screen will appear with fields for different accounts.

 

Click into the Amount fields for each account you want to post money into, and type in how much money you want to post into that account.

 

The Total field will display the amount you have posted so far

 

The Total Amount Tagged field will display the amount which you tagged to be posted

 

The Left To Post field will display the amount you have left to post in the different accounts.

 

The system will notify you if you enter amounts which exceed the amount you have left to post.

 

Once you have completed entering the amounts into the accounts, click on the Process button (click the Close button if you donwish to process it).

 

A Process Completed window will appear. Click the OK button.

 

The screen will then clear, and the entries will now appear in the Posted section.

 

PLEASE NOTE: Once an entry is in the posted it cannot be unposted.

 

The TOGGLE, REVERSE and SELECT ALL buttons all work the same in Posted as they did in UnPosted.

 

 

Deposit Rules

 

There are some specific rules that determine how the deposit screen gathers and processes its information. If a cheque number is the same for the same day then the system assumes that even though there is more than one entry for the cheque number on that day there was really only 1 cheque and so it adds up all of the entries and makes it one cheque to be deposited. This allows you to receive 1 cheque which is used to pay more than one account and have the deposit screen process it as 1 cheque. This may occur if for example some pays a cheque to cover both his own account as well as his wifeor childaccount. You need to record 2 or more payments but the deposit sees it as 1 cheque. If you see a deposit which says many then this is what has happened. This will cause a problem if you do not use correct chq numbers and just use a fullstop or some other character or name against the cheques. One other advantage to this is that if you use a cheque number called EFT for all of your manual credit card transactions then they will be treated as 1total amount as they are on most deposit slips. When used correctly this is a good feature. However, if not fully understood it can be a nightmare to some users.

 

Note that any entries classed as CASH or any entries journalled into the 1010 account with no method will be treated as CASH even if it wasnactually cash.

 

Dissect Button.

 

 

The dissect button allows you to dissect a deposit into multiple bank accounts. However, the deposit slip will not be usable as you can only print a deposit slip valid for a single account. If you decide to dissect a deposit then you must enter the amount of money beside each account that is to be deposited into that account. The total amount allocated must equal the deposited amount.
