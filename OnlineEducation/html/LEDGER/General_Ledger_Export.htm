General Ledger Export

 

GENERAL LEDGER EXPORT

 

If you are currently licensed for the General Ledger Export module then you can export your financial ledger data into either MYOB, Handisoft or Solution 6 which are all common programs that many accountants would use to process your financial details. If you provide the information in these formats, most accountants can read that data directly into their system and save time manually imputing balance and transaction figures. Your accountant can then manipulate these figures to produce your final accounts and provide you with adjusting journals to make your AM-Win accounts match the amendments your accountant has made.

 

The General Ledger Export module can be accessed from an icon in your AM-Win folder either on your desktop or from the Start button>All Programs>AMWIN>Ledger Export.

 

When you enter into the Ledger Export module there are several steps required before you can successfully export the data into an ASCII file.

 

1. You must first allocate each of your AM-Win Ledger accounts against an equivalent MYOB, Handisoft or Solution 6 ledger account. This is done by entering the alternate account number in the General Ledger Masterfiles in Am-win.

 

2. Go into the Ledger Export program and under the Conversion menu, select the option to Refresh the Intermediate file. This will create an intermediate file from your General Ledger Masterfiles so that the export program knows what MYOB account to use when exporting your financial data.

 

3.  Then again, under the Conversion menu, you can select the Intermediate File option to scroll through the file to ensure that every ledger has been linked and the link is accurate.

 

You are also able to print the Intermediate File using the Intermediate File Report option.

 

4. Before exporting the data from AM-WIN to your alternate accountantsoftware, you must confirm that the Trial Balance, balances and is correct.

 

NOTE: If the Trial Balance does not balance, then the system will export invalid data.

 

5. Select the Export option under the Conversion menu and then select the appropriate software you wish to export to.

 

NOTE: Solution 6 uses the MYOB export option.

 

6. You are then required to select the year and the month of the Financial Data you wish to be exported. Usually you would export the current monthdata.

 

7 You then have the option to export either Ledger Balances (MYOB.DAT), Ledger Transactions (MOYB_TR.DAT) or Sales Transactions (Service Invoices) (MYOB_SL.DAT) from the drop down box menus.

 

Ledger balances only export the balances for each Ledger Account. Ledger Transactions export all of the transactions that have taken place for each account for the period selected. Sales Transactions export all Service Invoices/Sales out to the relevant chosen software.

 

The exported file will usually be located in the Data directory under AM-Win (This is usually your Data1 folder.)

 

You can then import this file into the appropriate software or alternatively email or otherwise provide a copy this file to your accountant on a disk or other suitable media.

 

PLEASE NOTE: This process should only be performed once you have completed posting all entries for that period including any back posting that was required. To be safe you should close your GST period prior to exporting this data to ensure no more entries are back posted. Also note that if you provide your accountant with a backup of your data then please ensure you also retain a backup for yourself just in case the copy provided to your accountant is lost or becomes damaged.

 

IMPORTANT NOTE: MYOB is date dependent and therefore it will consider transactions dated in a certain period as belonging to that date. However, AM-win is NOT date dependant and transactions in a certain period can be dated outside of that period. To resolve this conflict, when the data is exported, any date outside the period is changed to be either the 1st or last day of that period. The data in AM-win is unchanged and only the export file is altered.

 

Importing to MYOB

If you plan to import the exported data into MYOB then the following process must be used.

 

If you are importing either the Ledger Balances or the Transaction Balances, then you must import the file into MYOB as a Transaction Journal. However, if you are importing Sales Transactions into MYOB then you must import the records as Sales Journals.

 

When importing the Sales Journals you can use the MATCH ALL option to match all MYOB fields to the AM-Win exported data.

 

If you are importing the Ledger Balances or Ledger Transactions, then you need to match the appropriate fields between AM-Win and MYOB.

 

These fields are matched as follows:

 

AM-Win    = MYOB

Transaction number = Transaction number

Date    = Date

Description  = Memo

First Amount  = Debit

Second Amount  = Credit

 

Errors

The most common error found in Exporting data from AM-Win to MYOB is that an account in MYOB that is linked from AM-win does not actually exist in MYOB.

 

Another common error is that a transaction, for some reason in AM-Win, does not balance. (The Transaction Audit should identify this transaction).

 

If you have ledgers in MYOB which are add only, these ledgers cannot have transactions, so these ledgers cannot be used as a link to AM-Win Ledgers. The import into MYOB will not work if these ledgers are linked to AM-Win Ledgers.

 

As from MYOB Version 13 onward, MYOB ledgers can no longer have dashes in the ledger codes. The import into MYOB will not work. You will need to correct the ledger codes in the Ledger Export, and Alternate Code in the General Ledger Masterfile.

 

In MYOB Version 12 and prior, the dashes are still required. In later versions, the dashes must be removed in all account numbers.
