Adding Item Details

 

ADDING ITEM DETAILS

 

Click on the Insert button.

 

DESCRIPTION TAB

You are required to either enter the Item Number for the item or allow the software to automatically assign the next item number. This field can also be used as a stock number/serial number etc. The number is auto generating if nothing is entered, but it also allows for alphanumeric characters to be inserted manually. This number will remain with the item regardless of the registration or other information changes which occur with this Item unless an item code change is done at a later time.

 

You are then able to enter a Rego Number if the Item is registered such as with a motor vehicle or some forklifts or other similar items. The rego number can be changed at any time and is not critical for tracking the Item.

 

You are also able to include a fleet number in the event that the item has been allocated one. The Manufactured or Build date can also be supplied for the Item. This date cannot be greater than the current entry date.

 

Item Category - there are 3 types of item category which determine how the Item is used and in what areas of the Item management system the Items can be accessed. These categories are explained in the Insert/Change Item section.

 

Condition  is the condition of the item, whether it be New, Used, Traded, or on Consignment.

 

Item Type  is a type of item within the selected category. For example within the for hire category you may have an item type of fork lift, bobcat, welder etc. In the computer category you may have an item type of computer, printer, photocopier, fax machine etc. You can insert new item types by clicking on the insert button from the item type attributes window.

 

Status  the status determines whether the Item is Catalog, Active, Inactive or Sold. Only active Items can be sold, hired or serviced. Catalog items are not relevant in the Hire process, so this option would not be used for hire equipment.

 

Serial No  if the item has a Serial Number, it may be added to this field.

 

Make and Model  if you click on the Make and Model button, the Make/Model screen will appear for you to select the required Make and Model of the item. New Makes and Models may be entered into the screen at this time also.

You are also able to manually type in a Make and Model into the fields, however, it is recommended for consistency that you use it from or add it to the Make and Model list.

 

Service Due button  if you click on the Service Due button, the Service Due screen will appear showing the last service which was performed on the item and any future services required. You may enter further future work which needs to be performed on the item on this screen also. Warranty information may also be entered onto this screen. This will be reflected also in the Job Item masterfile in the Service Log tab.

 

THE FOLLOWING LABELS ARE READ FROM THE JOB ITEM FILE SETUP AND WILL DIFFER BETWEEN INDUSTRIES.

 

Chassis No  enter the Chassis Number of the Item if applicable, or this may be Serial No depending on the label you have setup in the Job Item Masterfile. For a boat it would be a Hin number

 

Job button  when you click on the Job button, you will be taken to the job invoicing screen with the header already filled in with the required details, ready for you to enter parts etc. The invoice will automatically pick up the Customer and Bill To codes from the Item Register.

Engine No  enter the Engine Number of the Item if applicable. This may be a Motor Type for boats, depending on the label you have setup in the Job Item Masterfile.

 

Cyl  enter the number of Cylinders for the Item if applicable. This may be Weight for boats, depending on the label you have setup in the Job Item Masterfile.

 

Colour  by clicking on the down arrow next to this field you can select the colour of the Item if applicable. Further colours can be added to the system through Others>Maintenance>Attributes.

 

Engine CC  enter the Engine CC for the Item if applicable. This may be Trim for boats, depending on the label you have setup in the Job Item Masterfile.

 

Key No  enter the Key No for the Item if applicable. This may be Length for boats, depending on the lable you have setup in the Job Item Masterfile.

 

TransT  enter the Transmission Type for the Item if applicable. A new Transmission Type can be entered into the screen and selected, if required. This may be Starter for boats, depending on the label you have setup in the Job Item Masterfile.

 

Owner  The owner of the Item is used in the event that the Item is for sale or hire on a consignment basis. In this case service and repair work can be charged to the owner if required. The owner and Billto defaults to the owner and bill to codes.

 

Bill to  The bill to field indicates the customer or entity that will be billed for any work done against this Item. The bill to could be an internal customer or an owner of the Item where the Item could be on consignment and any charges for repair and maintenance of the Item will be charged to the entity who owns the Item.

 

RIGHT HAND SIDE OF SCREEN

NOTE: THESE LABELS ARE DEPENDENT ON THE INDUSTRY AND SETUP UNDER PROGRAMS>INDUSTRY FROM THE SALES MODULE. YOUR OWN HEADING SHOULD BE SETUP IN THIS AREA.

 

The right hand side of the Description Tab shows the specifications which are specific to the industry the Item belongs to. Each Item can be assigned as belonging to a different industry. For example a computer can be assigned a computer industry specs, a car can be assigned an automotive industry spec and a boat can be assigned a marine industry spec. The field label headings are then automatically changed to be consistent with labels and information that suit the chosen industry.

 

By clicking on the Industry button you can assign the industry to the chosen Item. The system will automatically use the first industry (Number 1) as the default industry if the specific industry has not yet been assigned. If the required industry has not yet been configured then you can select the industry option from the menu to insert a new industry and setup the appropriate field labels. There are 16 fields available for specifications of the Item. 14 of the fields are 10 characters long and the 15th and 16th fields are both 30 characters long.

 

Is Attachment  if this Item has been attached to another Item, this box will be automatically ticked.

 

Attached to - the Attached to field shows you the Item that the selected Item is attached to.

 

Acquired Date  indicates the date that the Item was acquired. This date can not be changed once it has been entered and saved. Usually it is updated at the time of purchase or trade-in but can also be entered when adding a new Item.

 

Default Location  this field holds the default location of the Item. This location could be a site location, floor location such as 1st Floor, 2nd Floor etc, or a department location such as sales, workshop, marketing, admin, etc. You must be licenced for Multi Location for this field to be applicable.

 

The Notes tab allows you to record misc notes about the selected Item. If notes have been entered previously then a green tick will indicate that notes are available. Otherwise a red cross will indicate that no notes have been entered for the selected Item.

 

ATTACHMENTS TAB

The attachments tab records the details of all of the Items which are attached to this Item. In the Description Tab of the item there is a field tick box called is attachment. If ticked then it indicates that this item is attached to the main Item item. For example the main Item may be a forklift and the attached Item in that case could be a set of side forks or a fork extension. If the Item was a boat then the attachment could be a motor or trailer, or both.

 

You can only attach an attachment to an Item if the attachment is not already attached to another Item. As such to attach that Item means you have to remove it from the Item it is currently attached to. You can then click in the insert key and this will allow you to attach any currently unattached attachments to the selected Item. You can have as many attachments as you wish attached to an Item. When the Item is hired out the attachments are hired out with it and the charge rate for the attachment is added to the Item hire charges. You should be aware of this when checking why an Item is being hired at a higher rate than the Item itself is setup for. If you wish to include the cost of the attachment in the Item hire rate then leave the hire rate of the attachment as zero.

 

When an attachment is attached to an Item then sometimes the performance of the Item is in some ways increased. In the case of a forklift the maximum lift weight, lift height etc may be increased or even decreased depending on the attachment. The attachment ratings are shown in the attachment tab along with the purchase price, on costs and recovery costs of the attachments.

 

INVOICING TAB  (THIS IS NOT APPLICABLE TO HIRE)

The invoicing tab contains information directly related to the sale of the Item. The Invoicing Tab is only available if the selected item is a For Sale item. Information would be relevant in this area if the Hire equipment became available for sale. This equipment would then be sold through the Sales Module.

 

Disposal Details - if the selected item has been sold or otherwise disposed of, then the customer field will show who the item was sold to as well as the date it was sold, the kilometres or hour reading at the time of sale, the method of disposal, the invoice number, sale amount without GST and the GST value.

 

The Accessories area at the top left of the screen allows you to enter any accessories that are fitted to the Item. These accessories are not to be confused with attachments, which are other Items attached to this Item. Accessories are generally either spare parts/stock items or revenue items which are available as accessories. They may include CD players or mag wheels for cars or lifejackets and fish finders for boats or any other accessory attached to the equipment being sold. Parts can be added as an accessory using the parts button or revenue items can be added using the revenue button. When an accessory is added, it can then be either changed or deleted using the change or delete buttons after highlighting the required record. You can also use the kit button to import parts and revenue item from a kit such as a safety or accessory kit. This allows you to record accessories that may or may not be fitted to the selected Item and then tag only the items that are actually fitted. When the item is then selected into a sales package, the accessories listed which are tagged will automatically be inserted as part of the sales quotation/order.

 

The Create Order button allows you to create a new sales order directly from the Item screen without going through the usual menu option.

 

The On Cost area allows you to enter on costs such as registration, insurance, dealer delivery and other costs to the Item. You have the choice of having the profit for the on costs show in the profit of the sale in the Finance screen, or not. This is a flag to be set. Refer to your local support person for this flag to be set if you do no wish to have the profit reflected in the profit for the sale. In some cases, the profit of On Costs will not be included in the salespersons commission, so in this case the flag should be set.

 

The Sale Price of the Item is the usual sale price including GST of the selected Item and is the price used when a sales quotation/order is produced. This price can be modified at the time of the sale with certain limitations based on the minimum sale price and managerprice.

 

The Attachments Price shows the total value of any attachments that are attached to the selected Item. The attachments may have a separate price attached to the Item which is added to the Item value or may have a nil price so that the attachment is inclusive of the sale price of the Item.

 

The Min Sale Price is the minimum price a salesperson with a standard password level can sell an Item for. The sales module will not allow such a person to sell the Item for less than the minimum sale price.

 

The ManagerPrice is the minimum price a Manager with a higher password level can sell the Item for through the sales module. The Managerprice is usually lower or equal to the minimum sale price.

 

The Fitted Accessories field shows the value of any tagged (having a green tick next to them) accessories and is included in the total price.

 

The Fitted On Costs field shows the value of any tagged (having a green tick next to them) on costs and is included in the total price.

 

The Total Price field includes the total of the sale price, attachments, fitted accessories and fitted on costs.

 

The Trade/Fleet discount field allows you to enter a standard discount for this Item and such discount is transferred to the sales module when this Item is selected through that module. The trade discount is taken into account when minimum sale price and minimum managerprices are used and the sale price less the discount cannot be less than the minimum sale price when the Item is sold unless this price is overwritten by a system supervisor\administrator (usually the business owner).

 

HIRE TAB

Daily Hire Rates can be set to Monthly, Weekly, Daily or Hourly Hire rates. Input the relevant rates. Note, these prices are Ex Gst.

 

Wet Hire Rates are applicable if an operator is to be made available with the Hire equipment. Input the relevant rates.

 

Insurance Recovery % requires a percentage to be entered for the system to calculate the Insurance Recovery Percentage on the invoice, if this is applicable.

 

Run Rate relates to an extra charge which may be applicable, and is used in conjunction with the Rate Type.

 

Fuel Rate relates to the fuel consumable rate, which is then used for a calculation in conjunction with the Rate Type. For example, there may be a charge of 50 cents per km or $ 1.50 per hour. This will then be automatically calculated on the Hire invoice.

 

Allowed relates to the free portion of the Run or Fuel rate, eg., the first 100km or first 5 hrs may be free before charges are applied.

 

Operator is the person who would be the default Operator of the Equipment when hired, if applicable.

 

Pay Rate would be the hourly rate that you pay the Contractor that is working for you and Est Hrs will then be calculated according to the hourly rate entered and the estimated number of hours for the Hire.

 

Check List - when you click on the Check List button, you can create a checklist for what is required to be checked prior to the Equipment going out for hire. These checklists are setup as Kits and can be created from Programs>Kits from the Sales Module.

 

The Availability button will bring up a screen to show the availability of the Item.

 

On the Plant Availability screen, the green section shows when the Item is available and the red section shows when the Item is unavailable.

 

The right hand side of the screen shows details of the current hire, if applicable, and any hire contracts which have been made for this Item.

 

To create a new Booking, click on the New Booking button. We will get into this further later in the manual.

 

LEDGER/EXPENSES TAB

 

Item Account  the Item account records the Item account number to be used to record the value of the Item. In the event of the Item being on hire, lease or consignment then it will have no Item value as you donactually own the Item. The number is automatically assigned when the Item category is selected. You can set the default ledger codes from the Ledger Setup area.

 

Accumulative Depreciation  is the Item depreciation account where the depreciation of the Item value is stored.

 

Depreciation Expense - is the expense side (debit) for the depreciation of the Item whereby the credit is the accumulative depreciation account.

 

Income  the income account is used to record the value of all income for that Item. This could be hire income, charge out income, sales income etc. The income number is automatically assigned when the Item category is selected. You can set the default ledger codes from the Ledger Setup area.

 

Parts Expense  the parts expense account is used to record the cost of any parts that are used in the service and maintenance of the Item. The expense number is automatically assigned when the Item category is selected. You can set the default ledger codes from the Ledger Setup area.

 

Lab Expense  the labour expense account is used to record the cost of any labour that is used in the service and maintenance of the Item. The expense number is automatically assigned when the Item category is selected. You can set the default ledger codes from the Ledger Setup area.

 

Rev Expense - the revenue expense account is used to record the cost of any misc expenses that are used in the service and maintenance of the Item. This includes registration, insurances, sublet costs for repair and services etc. The expense number is automatically assigned when the Item category is selected. You can set the default ledger codes from the Ledger Setup area.

Insurance Recovery - is the account where the insurance depreciation levy percentage is stored for the recovery of insurance and other overheads against a hire.

 

Sales Cost - is the cost of sales of an item which is calculated based on the purchase price of the item less any accumulative depreciation or based on a re-evaluation of the Item. This cost does not include workshop maintenance and other direct costs which are associated with the item.

 

EXPENSE LISTBOX

The expense listbox is a list of direct expenses, which have been allocated to the item through either a purchase, purchase order or through a cheque entry process.

 

The icon to the right of the sales cost button allows you to view any lease contracts that are allocated to this item. The icon is only available if a contract for this item exists.

 

ACCOUNTS TAB

This tab holds the financial details of the Item including Depreciation information and Income and Expense information on the Item.

 

Purchase price (Ex) - the Purchase Price (Ex) is the price that this Item was purchased for or in the event that it is a catalogue item then it is the list price of the catalogue item.

 

Acquired Method  The purchase method will either be Hire Purchase, Personal Loan, Lease, Floorplan, Trade-In or Purchase. If the item was traded in from a private person with no ABN number then the second hand tax rule applies whereby GST of 1/11th of the trade-in value can be claimed back when the Item is sold. However, if the Item was traded in by a company then the tax office requires that a purchase is made and as such the item needs to be purchased rather than traded in. Depending on how the Item was entered into the system, this field will be autofilled.

 

FLOORPLAN DETAILS

The Floorplan section is not applicable to Hire.

 

DEPRECIATION

The use of the Depreciation Tab is only applicable for Items which are of an Internal Status eg., your own business vehicles, computers.

 

Depreciation Schedule - this button produces a depreciation schedule for the selected Item. The screen shows 2 tabs. The Actual tab which show the actual depreciation that has been posted to the ledgers on a monthly basis. The Projected tab displays a calculation based on the depreciation rate, claim rate and written down value showing the expected future depreciation figures based on the current settings.

Written Down Value - this field contains the initial value for the start of the depreciation schedule and can be entered via the Supervisor menu in the Depreciation Section Opening Balance.

 

The % rate is the rate that the selected Item depreciates at.

 

For months - this field is used to store the number of months that the item is depreciated over. For example if the item depreciates over 4 years then the 48th month (last month) will have the item depreciate at the remaining value of the item. As such the item will be fully depreciated after 48 months. The depreciation amount is calculated on a monthly basis, based on the total yearly depreciation divided by 12 and the depreciation value will only change at the beginning of a new financial year.

 

Write off at last Month  this tick box means that the full remaining depreciation value for the Item will be written off at the last month of the depreciation period.

 

Claimed Percent  this field stores the percentage that can be claimed for business purposes. The Item may have 20% personal use and as such the claim percent will be 80%.

 

Depreciation Method - determines either Straightline or Diminishing depreciation. Straightline means that the depreciation is a percentage of the original value of the Item. This means that if the Item is depreciated at 20% then after 5 years the item will be fully depreciated. However, Diminishing means that the Item is depreciated at a diminishing written down value. The written down value is updated at the end of each financial year.

 

Next Due  this date shows the date that the depreciation is next due on. The depreciation is processed on a monthly basis during the month end processing.

 

Initial Item Value - is the original purchase price of the Item. This field is automatically filled in.

 

Depreciation value YTD - records the amount of depreciation for the Item for the current financial year. This field is automatically filled in.

 

Depreciation Value Total - this value records the amount of depreciation for the Item for the life of the Item. This field is automatically filled in.

Trade in and Purchase Details

 

If this item was a trade in Item, then the details recorded at the time of trade in such as the trading type, valuation, revs details and acquired details are recorded here. This information is used for the Police Book and also for calculating GST on sales.

 

The tab also details the income and all expenses that have been allocated to that Item as well as calculating the profitability of the Item both on a cashflow and a book keeping basis.

 

FINANCE CHECK

When an Item is traded, generally in the case of motor vehicles, you are able to do a check to see if there is any finance still owing on the Item. The contact details for this check would be different between states, however the details of the revs check may be recorded in the Financial Check area. This information is autofilled from the details entered from the check in the Sales Module.

 

ACQUIRED DETAILS

The Acquired Details show details of the purchase made for the Item.

 

FINANCIAL DETAILS

There are 2 methods of calculating profit/loss in the system. There is the cash flow method often preferred by business and operation managers and the Book keeping method usually used by accountants.

 

Both methods use the total income as the addition of the Hire/Charge income, Extra Income and the Sold for income.

 

The Cashflow Method determines the profit of an Item based on the Total Income less the day to day costs of maintaining the Item. These costs depending on the finance type include the Acquisition Cost, the maintenance costs and the lease/loan repayment charges. The depreciation of the Item is not taken into consideration.

 

The Book Keeping Method determines the profit of an Item based on the Total Income less the cost of the Item including depreciation costs. In this case the cost is calculated based on the depreciation value of the Item in the case of an Item which is on Hire Purchase or a loan or a straight purchase. In the case of a lease or rental agreement the cost is still calculated on the payments made as these Items do not get depreciated. Maintenance and Term charge costs are also calculated.

 

INCOME

The Hire Income shows the income generated from either hiring or charging out of an Item.

 

The Extra Income shows the income that is generated from the sale of parts, labour and revenue items as a part of the hire or charge out.

 

The Hire Insurance income shows the income that has been derived from the hire insurance levy charges.

 

Sold For is the amount of income that is generated from the sale of the Item.

Total Income is the addition of the Hire/Charge income, the extra income and the sold for income.

 

EXPENSES

Acquisition Cost is the cost of purchasing or otherwise acquiring the Item.

 

Workshop Charges relates to the charge of work performed on the Item through the workshop. These are ex gst figures

 

Other Expenses relate to other charges relating to the Item, eg., fuel, tyres etc. These might be charges entered through the Cheque Entry screen that do not relate to work entered on the workshop job. These are ex gst figures

 

Lease/Hp Expense is the value of lease or HP payments made for the selected item.

 

Term charges/Floorplan Charges is the cost of establishment fees, loan fees and other finance charges over and above the usual repayments.

This is not applicable for Hire.

 

Commission is the amount of commission paid to the sales person who sold the Item.

 

Overtrade is the value of any overtrade amount in the event that this was a traded item. If the item was valued at $5000 and a trade in of $6000 was given to the customer then the item would be considered to have an overtrade value of $1000 as it was only worth $5000 although $6000 was paid for it.

 

Total Expense is all of the expense items added together depending on the Profit Method chosen.

 

Profit / Loss is the profit or loss based on the Total Income less the Total Expense. This value will change depending on the Profit Method being used.

 

BUTTONS

The Filter button allows you to filter the Item list even further to include only Item types in the list. After pressing the Filter Button you can then select the Item Category and the Item Type, which will refine the Item list to only those Items that match the filter criteria. You can also filter by choosing a particular Status, Condition and Make and Model. You can also filter the Item list by using a range within the first 5 specification of the selected Item.

 

NOTE: Sorting the Item list. You can sort the Item list any of the column headings. Clicking on the same heading again, will reverse the order of the sort.

 

The Supervisor button allows the user to change many of the otherwise unavailable fields for the selected Item. These fields include opening balances, depreciation figures, payments details etc. These fields are available when you first insert an Item but not when you revise one other than by using the Supervisor button. This button is only available to users who have a password access level of 80 or above.

 

The Clone button allows you to clone the existing selected Item and create a new Item using similar details. The make, model, specifications, ledgers assignments and many other details will be cloned from the selected Item. This saves you having to retype the same details again and again if you have multiples of the same Item. However, the payment, some depreciation and other accounting fields are not cloned to avoid the possibility of creating incorrect accounting transactions. You are also required to enter some unique fields some as the Item Number, Rego Number and Fleet number etc.

 

The Insert button allows you to add a new Item by clicking on the Insert button.

 

The Status Filter drop down menu allows you to filter Items by status. If you select a status such as Active, Inactive etc then the system will display only Items that match this criteria.

 

The Website Upload button is available as a direct link to the Trading Post. You must have an agreement setup with the Trading Post. This option creates a file for the highlighted Item which uploads directly to the Trading Post. If you have a picture associated with the Item, the file name of the picture must be the same as the Item number.

 

The Condition Filter drop down menu allows you to select a Condition of New, Used, Trd or Consignment and will then only show Items that fit this criteria. The status and condition filters can be used together to only show Items that meet both criteria.

 

The Clear Search button clears the search criteria filters and reloads all of the Items back into the list.

 

Picture File button  This button allows you to add in up to 6 different pictures linked to an Item. This is helpful when looking at Items for sale or for hire. To add a picture, click on the picture button. You can then click on the Picture File button at the bottom of the screen to add a picture to each picture tab. There are 6 picture tabs which contain 1 picture per tab. You can also enlarge the selected picture by clicking on the Enlarge button at the bottom of the screen. The picture contained in the Picture 1 tab is displayed at the top right hand corner of the Item register screen. Click on the Picture File button to select the data path per picture.

 

The Item Stock button shows a report of all Items showing Make/Model, Chassis number etc, and also the selling price and status of the Item.

 

The Cost Price button shows a report similar to the above report, but showing cost pricing, but with more refined filtering as shown below.
